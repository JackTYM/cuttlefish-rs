# API Reference

Cuttlefish provides REST and WebSocket APIs for programmatic access to the agent system.

## Authentication

All requests require an API key in the `Authorization` header:

```bash
Authorization: Bearer YOUR_API_KEY
```

Set `CUTTLEFISH_API_KEY` environment variable to configure your API key.

## REST Endpoints

### Health Check

```
GET /health
```

Returns server status:

```json
{
  "status": "ok",
  "version": "1.0.0"
}
```

### Projects

#### List Projects
```
GET /api/projects
```

#### Create Project
```
POST /api/projects
Content-Type: application/json

{
  "name": "my-project",
  "description": "Project description",
  "template": "rust-cli"
}
```

#### Get Project
```
GET /api/projects/:id
```

#### Delete Project
```
DELETE /api/projects/:id
```

### Templates

#### List Templates
```
GET /api/templates
```

#### Validate Template
```
POST /api/templates/validate
Content-Type: application/json

{
  "content": "template markdown content"
}
```

### Agents

#### Send Request
```
POST /api/agents/request
Content-Type: application/json

{
  "project_id": "project-123",
  "message": "Add a new feature",
  "context": {}
}
```

#### Get Status
```
GET /api/agents/status/:request_id
```

## WebSocket

Connect to `ws://localhost:8080/ws` for real-time updates.

### Connection

```javascript
const ws = new WebSocket('ws://localhost:8080/ws', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});
```

### Message Types

#### Chat Message
```json
{
  "type": "chat",
  "project_id": "project-123",
  "message": "What should I do?"
}
```

#### Status Update
```json
{
  "type": "status",
  "project_id": "project-123",
  "status": "running",
  "agent": "coder"
}
```

#### Log Update
```json
{
  "type": "log",
  "project_id": "project-123",
  "level": "info",
  "message": "Building project..."
}
```

#### Error
```json
{
  "type": "error",
  "project_id": "project-123",
  "error": "Build failed"
}
```

## Error Responses

All errors follow this format:

```json
{
  "error": "error_code",
  "message": "Human-readable error message",
  "details": {}
}
```

### Common Error Codes

| Code | Status | Meaning |
|------|--------|---------|
| `unauthorized` | 401 | Invalid or missing API key |
| `not_found` | 404 | Resource not found |
| `invalid_request` | 400 | Invalid request parameters |
| `internal_error` | 500 | Server error |
| `sandbox_error` | 503 | Docker sandbox unavailable |

## Rate Limiting

- **Requests**: 100 per minute per API key
- **WebSocket**: 1 connection per API key
- **Concurrent agents**: Limited by sandbox configuration

## Examples

### Create and Run a Project

```bash
# Create project
curl -X POST http://localhost:8080/api/projects \
  -H "Authorization: Bearer $CUTTLEFISH_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-app",
    "template": "rust-cli"
  }'

# Send request to agent
curl -X POST http://localhost:8080/api/agents/request \
  -H "Authorization: Bearer $CUTTLEFISH_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "my-app",
    "message": "Add a hello command"
  }'
```

### WebSocket Real-Time Updates

```javascript
const ws = new WebSocket('ws://localhost:8080/ws');

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  
  if (message.type === 'log') {
    console.log(`[${message.level}] ${message.message}`);
  } else if (message.type === 'status') {
    console.log(`Status: ${message.status} (${message.agent})`);
  }
};

// Send a message
ws.send(JSON.stringify({
  type: 'chat',
  project_id: 'my-app',
  message: 'Build the project'
}));
```
