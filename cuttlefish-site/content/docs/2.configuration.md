# Configuration

Cuttlefish uses TOML configuration files with environment variable overrides for secrets.

## Config File Locations

Configuration is loaded in this order (first found wins):

1. `cuttlefish.toml` in current directory
2. `/etc/cuttlefish/cuttlefish.toml` (system-wide)
3. `~/.config/cuttlefish/config.toml` (user)

## Server Configuration

```toml
[server]
host = "127.0.0.1"      # Bind address (use 0.0.0.0 for external access)
port = 8080             # HTTP/WebSocket port
```

## Database Configuration

```toml
[database]
path = "cuttlefish.db"  # SQLite database path (WAL mode enabled)
```

## Docker Sandbox Configuration

```toml
[sandbox]
docker_socket = "unix:///var/run/docker.sock"
memory_limit_mb = 2048  # Per-container memory limit
cpu_limit = 2.0         # Per-container CPU limit (cores)
disk_limit_gb = 10      # Per-container disk limit
max_concurrent = 5      # Maximum concurrent sandboxes
```

## Model Providers

### AWS Bedrock (Recommended)

```toml
[providers.claude]
provider_type = "bedrock"
model = "anthropic.claude-3-5-sonnet-20241022-v2:0"
region = "us-east-1"

[providers.claude-fast]
provider_type = "bedrock"
model = "anthropic.claude-3-haiku-20240307-v1:0"
region = "us-east-1"

[providers.claude-opus]
provider_type = "bedrock"
model = "anthropic.claude-3-opus-20240229-v1:0"
region = "us-east-1"
```

## Agent Configuration

Map agents to model categories:

```toml
[agents.orchestrator]
category = "deep"               # Uses claude (Sonnet)

[agents.coder]
category = "deep"               # Uses claude (Sonnet)

[agents.critic]
category = "unspecified-high"   # Can use a different model
```

## Environment Variables

**Required:**
```bash
CUTTLEFISH_API_KEY      # API key for WebUI/TUI authentication
```

**For AWS Bedrock:**
```bash
AWS_ACCESS_KEY_ID       # AWS credentials
AWS_SECRET_ACCESS_KEY
AWS_DEFAULT_REGION      # e.g., us-east-1
```

**For Discord:**
```bash
DISCORD_BOT_TOKEN       # Discord bot token
```

## Model Categories

| Category | Use Case | Recommended Model |
|----------|----------|-------------------|
| `deep` | Complex reasoning, autonomous execution | Claude 3.5 Sonnet, Claude 3 Opus |
| `quick` | Simple tasks, fast responses | Claude 3 Haiku |
| `ultrabrain` | Hard logic, architecture decisions | Claude 3 Opus |
| `visual` | Frontend, UI/UX, design work | Claude 3.5 Sonnet |
| `unspecified-high` | General tasks, higher effort | Claude 3.5 Sonnet |
| `unspecified-low` | General tasks, lower effort | Claude 3 Haiku |
