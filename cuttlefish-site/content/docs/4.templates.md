# Templates

Start new projects from proven templates. Templates are markdown files with YAML frontmatter that define project structure and initial configuration.

## Using Templates

```bash
# Create a new project from a template
cuttlefish new my-project --template rust-cli

# List available templates
cuttlefish templates list

# View template details
cuttlefish templates show rust-cli
```

## Available Templates

### Rust Templates

- **rust-cli**: Command-line application with argument parsing
- **rust-lib**: Library with tests and documentation
- **rust-web**: Axum web server with database integration

### Web Templates

- **nuxt-app**: Nuxt 3 web application with Tailwind CSS
- **react-app**: React 18 with TypeScript and Vite
- **vue-app**: Vue 3 with TypeScript and Vite

### Backend Templates

- **fastapi**: Python FastAPI backend with SQLAlchemy
- **nodejs-express**: Node.js Express server with TypeScript
- **go-gin**: Go Gin web framework

### Integration Templates

- **discord-bot**: Discord bot starter with slash commands
- **github-action**: GitHub Actions workflow template
- **docker-app**: Dockerized application with compose

## Template Structure

Templates are stored in `templates/` directory:

```
templates/
├── rust-cli.md
├── rust-lib.md
├── nuxt-app.md
└── ...
```

Each template contains:

```markdown
---
name: "Rust CLI"
description: "Command-line application"
language: "rust"
tags: ["cli", "rust"]
---

# Template Content

Instructions for using the template...
```

## Creating Custom Templates

1. Create a markdown file in `templates/`
2. Add YAML frontmatter with metadata
3. Document the project structure
4. Include setup instructions
5. Submit to marketplace (optional)

## Template Marketplace

Share your templates with the community:

```bash
# Publish template
cuttlefish templates publish my-template.md

# View published templates
cuttlefish templates marketplace
```

Templates are validated for:
- Correct YAML frontmatter
- Valid project structure
- Clear documentation
- Working examples
